# HDKala فروشگاه چندصفحه‌ای

HDKala اکنون به صورت یک وب‌سایت چندصفحه‌ای پیاده‌سازی شده است. هر صفحه HTML لایه‌ی نمایشی خود را دارد و اسکریپت‌های ماژولار ES آن را مدیریت می‌کنند. اجزاء مشترکی مانند هدر، فوتر و کارت محصول به صورت وب‌کامپوننت پیاده‌سازی شده‌اند تا در تمام صفحات یک‌دست باقی بمانند.

## ساختار پروژه

```
src/
  input.css          # ورودی Tailwind
  js/
    bootstrap.js     # ثبت کامپوننت‌ها و اعلان‌ها
    components/      # وب‌کامپوننت‌های مشترک (هدر، فوتر، کارت محصول)
    data/            # داده‌های نمونه و ثابت‌ها
    pages/           # اسکریپت اختصاصی هر صفحه
    services/        # ماژول‌های منطق (محصولات، سبد، علاقه‌مندی و ...)
    utils/           # ابزارهای کمکی مانند مدیریت مسیرها
  pages/
    index.html       # صفحه اصلی
    products/        # صفحات فهرست و جزئیات محصول
    auth/            # صفحات ورود و ثبت‌نام
    cart/, wishlist/, compare/
    checkout/, admin/, profile/, orders/, addresses/
    blog/, contact/, about/
```

پس از اجرای فرایند ساخت، پوشه‌ی `dist/` شامل همین ساختار به همراه فایل خروجی Tailwind (`output.css`) خواهد شد.

## اجرای توسعه و ساخت

```bash
# ساخت نسخه توسعه و پایش استایل‌ها
npm run dev

# ساخت نسخه نهایی (HTML + JS + CSS فشرده)
npm run build
```

فرمان‌های بالا ابتدا فایل‌های HTML/JS را به مسیر `dist/` کپی می‌کنند و سپس خروجی Tailwind را در همان مسیر تولید می‌کنند. برای مشاهده صفحات کافی است فایل‌های `dist/` را روی یک سرور استاتیک سرو کنید.

## یادداشت‌ها

- مسیرهای ناوبری به کمک ویژگی `data-base` روی تگ `<body>` برای هر صفحه تنظیم می‌شوند تا لینک‌ها در زیرمسیرها نیز درست کار کنند.
- وضعیت‌های سبد خرید، علاقه‌مندی و مقایسه در `localStorage` نگه‌داری می‌شود و بین صفحات به اشتراک گذاشته می‌شود.
- برای افزودن صفحه جدید، HTML آن را در `src/pages/` قرار دهید و در صورت نیاز ماژول جاوااسکریپت متناظر را داخل `src/js/pages/` بسازید، سپس `npm run build` را اجرا کنید.
